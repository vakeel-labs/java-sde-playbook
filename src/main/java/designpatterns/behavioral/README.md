# Behavioral Design Patterns

Behavioral patterns are concerned with algorithms and the assignment of responsibilities between objects. They describe patterns of communication between objects.

## Patterns

| Pattern | Use When |
|---------|----------|
| [Strategy](./01-strategy) | Swap algorithms at runtime |
| [Observer](./02-observer) | Notify multiple subscribers of state changes |
| [Command](./03-command) | Encapsulate requests; support undo/redo |
| [Iterator](./04-iterator) | Traverse a collection without exposing its structure |
| [Template Method](./05-template-method) | Define a skeleton algorithm; subclasses fill in steps |
| [State](./06-state) | Change behavior when internal state changes |
| [Chain of Responsibility](./07-chain-of-responsibility) | Pass request along a chain of handlers |
| [Mediator](./08-mediator) | Centralize complex communication between objects |
| [Memento](./09-memento) | Save and restore object state (undo) |
| [Visitor](./10-visitor) | Add new operations to objects without modifying them |
| [Interpreter](./11-interpreter) | Define grammar and interpret sentences |
