# NoSQL Databases

NoSQL databases provide flexible schemas and horizontal scalability for use cases where relational databases fall short — high write throughput, unstructured data, and massive scale.

## Why NoSQL?

- Schema flexibility (schema-less or schema-on-read)
- Horizontal scaling (sharding across nodes)
- High availability and fault tolerance
- Optimized for specific access patterns

## NoSQL Categories & Examples

| Type | Examples | Best For |
|------|----------|---------|
| Document | MongoDB | JSON-like data, user profiles, catalogs |
| Key-Value | Redis | Caching, sessions, leaderboards |
| Wide Column | Cassandra | Time-series, IoT, write-heavy workloads |
| Search Engine | Elasticsearch | Full-text search, log analytics |
