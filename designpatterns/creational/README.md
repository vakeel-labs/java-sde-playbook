# Creational Design Patterns

Creational patterns abstract the object instantiation process, making a system independent of how its objects are created, composed, and represented.

## Patterns

| Pattern | Use When |
|---------|----------|
| [Singleton](./01-singleton) | One instance needed globally (DB connection, config, logger) |
| [Factory Method](./02-factory-method) | Subclass decides which object to create |
| [Abstract Factory](./03-abstract-factory) | Create families of related objects (UI themes, DB drivers) |
| [Builder](./04-builder) | Complex object with many optional parameters |
| [Prototype](./05-prototype) | Clone objects when creation is expensive |
