# Design Patterns (GoF — Gang of Four)

All 23 classic design patterns from the "Design Patterns: Elements of Reusable Object-Oriented Software" book, implemented in Java. Design patterns are reusable solutions to commonly occurring problems in software design.

## Categories

| Category | Count | Purpose |
|----------|-------|---------|
| [Creational](./creational) | 5 | Object creation mechanisms |
| [Structural](./structural) | 7 | Class and object composition |
| [Behavioral](./behavioral) | 11 | Object collaboration and responsibilities |

## Creational Patterns

| # | Pattern | Intent |
|---|---------|--------|
| 01 | [Singleton](./creational/01-singleton) | Ensure only one instance exists |
| 02 | [Factory Method](./creational/02-factory-method) | Delegate object creation to subclasses |
| 03 | [Abstract Factory](./creational/03-abstract-factory) | Create families of related objects |
| 04 | [Builder](./creational/04-builder) | Construct complex objects step by step |
| 05 | [Prototype](./creational/05-prototype) | Clone existing objects |

## Structural Patterns

| # | Pattern | Intent |
|---|---------|--------|
| 01 | [Adapter](./structural/01-adapter) | Make incompatible interfaces work together |
| 02 | [Bridge](./structural/02-bridge) | Separate abstraction from implementation |
| 03 | [Composite](./structural/03-composite) | Treat individual objects and compositions uniformly |
| 04 | [Decorator](./structural/04-decorator) | Add behavior to objects dynamically |
| 05 | [Facade](./structural/05-facade) | Provide a simplified interface to a subsystem |
| 06 | [Flyweight](./structural/06-flyweight) | Share state to support large numbers of objects |
| 07 | [Proxy](./structural/07-proxy) | Provide a surrogate or placeholder for another object |

## Behavioral Patterns

| # | Pattern | Intent |
|---|---------|--------|
| 01 | [Strategy](./behavioral/01-strategy) | Define a family of interchangeable algorithms |
| 02 | [Observer](./behavioral/02-observer) | Notify dependents of state changes |
| 03 | [Command](./behavioral/03-command) | Encapsulate requests as objects |
| 04 | [Iterator](./behavioral/04-iterator) | Access elements sequentially without exposing internals |
| 05 | [Template Method](./behavioral/05-template-method) | Define a skeleton algorithm in a base class |
| 06 | [State](./behavioral/06-state) | Alter behavior when internal state changes |
| 07 | [Chain of Responsibility](./behavioral/07-chain-of-responsibility) | Pass requests along a chain of handlers |
| 08 | [Mediator](./behavioral/08-mediator) | Reduce chaotic dependencies between objects |
| 09 | [Memento](./behavioral/09-memento) | Capture and restore object state |
| 10 | [Visitor](./behavioral/10-visitor) | Add operations to objects without modifying them |
| 11 | [Interpreter](./behavioral/11-interpreter) | Define a grammar and interpret sentences |
