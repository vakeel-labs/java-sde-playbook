# Microservices Concepts

Core architectural patterns and principles behind building resilient, scalable, and maintainable microservices systems.

## Patterns

| Pattern | Description |
|---------|-------------|
| [Service Discovery](./service-discovery) | How services find each other dynamically |
| [API Gateway](./api-gateway) | Single entry point for all client requests |
| [Circuit Breaker](./circuit-breaker) | Prevent cascading failures |
| [Load Balancing](./load-balancing) | Distribute traffic across service instances |
| [Event-Driven Messaging](./event-driven-messaging) | Async communication via events |
| [Saga Pattern](./saga-pattern) | Distributed transactions across services |
| [CQRS & Event Sourcing](./cqrs-event-sourcing) | Separate reads/writes; store events not state |
| [Distributed Tracing](./distributed-tracing) | Track requests across service boundaries |
