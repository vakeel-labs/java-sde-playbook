# Low Level Design (LLD)

Low Level Design focuses on the object-oriented design of individual components — translating requirements into classes, interfaces, and their interactions.

## LLD Interview Framework

1. **Clarify requirements** — Functional (what it does) and non-functional (scale, constraints)
2. **Identify core entities** — What are the main objects/actors?
3. **Define relationships** — Inheritance, composition, association
4. **Apply SOLID principles** — Every design decision should have justification
5. **Apply design patterns** — Identify where patterns solve recurring problems
6. **Sketch the class diagram** — Classes, interfaces, fields, methods, relationships
7. **Walk through use cases** — Show how the design handles key flows

## Skills Tested

- OOPs mastery (abstraction, polymorphism, encapsulation)
- SOLID principles application
- Choosing the right design pattern
- Extensibility (adding features without breaking existing code)
- Code quality and naming
